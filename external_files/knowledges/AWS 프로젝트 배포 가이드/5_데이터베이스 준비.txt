[데이터베이스 준비]
- 온라인상 오픈된 데이터베이스를 준비하겠습니다.
    앞으로 개발할 모든 서비스는 이 데이터베이스를 사용할 것입니다.
    데이터베이스는 IT 서비스에 있어서 가장 중요한 자원인 데이터를 저장하는 공간이므로,
    개발시에만 사용되는 개발 데이터베이스,
    배포시에 사용되는 배포 데이터베이스로 2개로 나누어 생성하겠습니다.

- AWS RDS 생성법
    AWS RDS 클라우드 서비스를 이용한 데이터베이스를 생성할 것입니다.

    1. AWS 콘솔에서 Amazon RDS 를 누르고, 대시보드에서 데이터베이스 생성을 누릅니다.
        제 설정은,
        MySQL 1.0.35, 프리티어 템플릿, 자격증명 자체 관리, DB 인스턴스 db.t3.micro,
        스토리지 범용 SSD(gp2) 20GiB, 퍼블릭 엑세스 yes, VPC 보안그룹 새로 생성
        위와 같이 만들었습니다.
        중요한 것은,
        생성이 완료되고 외부에서 접근이 가능하면 됩니다.
        생성이 완료되면 데이터베이스 목록에서 해당 항목을 클릭하면 접속 가능한 엔드포인트 등,
        데이터베이스 정보를 확인할 수 있습니다.
        이런식으로 개발 데이터베이스와 배포 데이터베이스를 만들어줍시다.

    2. 접속 테스트 및 보안 설정
        데이터베이스 항목을 누르고,
        보안 - VPC 보안 그룹의 항목을 선택합니다.
        보안그룹 설정에서 인바운드 규칙을 누릅니다.
        이 규칙은 외부에서 이 서버로 접근할 때에 허용할 IP 와 금지할 IP 를 결정하는 설정으로,
        데이터베이스에 접근하는 주체는,
        개발자의 PC, 테스트 서버, 개발 서버만 접근 가능하면 됩니다.
        인바운드 규칙 편집을 눌러보면,
        인바운드 규칙을 추가할 수 있는데,
        규칙 추가 버튼을 누르고, 3306 포트를 입력하고,
        소스 부분에 접근을 허용할 IP 를 입력하면 됩니다.
        일단 본인의 IP 를 입력하고, 추후에 테스트 서버, 개발 서버의 IP 를 추가하면 됩니다.

        접속 테스트는,
        로컬에 설치된 Mysql workbench 등의 데이터베이스 관리툴을 사용하여,
        앞서 확인한 데이터베이스 인스턴스의 접근 엔드포인트로 접속이 잘 되는지를 확인하세요.

    2. 추후 정보 전달을 위한 문서화를 합시다.
        가이드 3번 글에서 만든 ServerInfo 폴더 안에,
        데이터베이스 정보.txt 파일을 만들고,
        이 안에,


        [Aws RDS]

        (database-dev)
        URL :
        DBMS : MySQL 8.0.35
        port : 3306
        id :
        pw :
        DB 인스턴스 : db.t3.micro
        스토리지 : 범용 SSD(gp2) 20GiB
        Vpc :

        (database-prod)
        URL :
        DBMS : MySQL 8.0.35
        port : 3306
        id :
        pw :
        DB 인스턴스 : db.t3.micro
        스토리지 : 범용 SSD(gp2) 20GiB
        Vpc :


        위와 같은 형식으로 데이터베이스 정보를 저장하면 됩니다.
